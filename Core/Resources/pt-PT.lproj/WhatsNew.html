Novidades no Growl 2.1

Alterações principais

  • Corrigimos os problemas com a acção do Automator, pelos quais pedimos desculpa. O Xcode tentou ajudar com as definições de validação do projecto, mas acabou por desligar algo que era necessário. Resolvemos isto permanentemente.
  • Resolvido problema com a detecção de protecção de ecrã e de bloqueio de ecrã para que funcione como esperado.

Novidades no Growl 2.1

Alterações principais

  • Adicionada a possibilidade de executar scripts como uma acção.
  • Adicionada a possibilidade de controlar notificações através de regras. Ver http://growl.info/documentation/applescript-rules para mais informação.
  • O Growl já não deixa de responder quando recebe um elevado número de notas (mais de 100) de uma só vez.
  • As apresentações Webkit podem agora ter botões de fecho personalizados com estilos.

Alterações gerais

  • Tornado mais claro quando a aplicação decide prioridades.
  • Aumento do tamanho da imagem no AppleScript de 128x128 para 1024x1024.
  • Ao utilizar encaminhamento em conjunto com encaminhamento para o Centro de notificações, agora verá notificações em ambos os computadores.
  • O plugin de acção Prowl agora envia notificações correctamente se tiver o clique em notificações definido.

Automator

  • A acção do Automator está agora traduzida.
  • A workflow GrowlAction irá agora ler da descrição da acção do Automator

Novidades no Growl 2

Geral

  • - Adiciona suporte para o Centro de notificações do OSX 10.8
  • Adiciona suporte para os serviços Prowl e Boxcar como acções.
  • O Agregador tem melhor detecção de inactividade
  • O Growl é agora executado numa sandbox.

Preferências

Aparências

  • Maior controlo com a Fala.
  • Separados os estilos de apresentação dos das acções.
  • Pode justificar texto à direita e à esquerda no estilo Music Video.
  • Resolve problemas de altura para notificações de múltiplas linhas com o estilo Music Video.
  • O estilo Nano pode agora apresentar notificações em múltiplos locais, esquerda, centro e direita.
  • Adicionada uma apresentação SoundAction.
  • Adicionada uma forma de não ter quaisquer notificações visuais.

Resolução de problemas

  • Resolvido o problema com a elevada utilização do CPU.
  • Resolvidas várias falhas.
  • Os atalhos de teclado podem agora ser removidos.

Alterações no Growl 1.4

Geral

  • Redesign do separador de Aplicações! Trata-se de uma grande reformulação, deverá ser muito mais fácil de utilizar.
  • Movidos controlos do Agregador para um separador à parte.
  • Grandes melhoramentos no suporte de rede.

Reorganização do Painel de preferências

  • Novo interruptor para botões I/O.
  • Adicionado suporte para teclado ao seleccionador de posição.
  • Reorganização do separador Geral
  • Separador Rede - Adicionado suporte para subcrições
  • Separador Histórico, menu de contexto na tabela do histórico para saltar para as definições da app/notificações.
  • O separador Agregador é a nova casa dos controlos e definições relacionadas com o Agregador
  • Adicionar uma nova apresentação agora não deverá necessitar que o Growl reinicie

Rede

  • Suporte para Notification-Callback-Target, um campo de URL no GNTP. Ao clicar, o Growl permite ao OS X decidir como abrir o URL (como por exemplo o browser predefinido para um url http:// )
  • Suporte para subscrições, subscrever todas as aplicações numa máquina remota.
  • - Melhoramentos gerais na autenticação de rede, removida a necessidade de, e a utilização de cabeçalhos de palavra-passe se o campo de palavra-passe estiver em branco.
  • Resolvida falha de rede relacionada com a resolução de dados de endereço.
  • Apenas vincular ao socket local se não estiver a escutar para recepção de notificações.
  • Cache de dados de endereços e monitorização de alterações na rede para limpar caches

Applescript

  • Suporte a resposta a cliques em URL, tal como Notification-Callback-Target no GNTP

Resolução de problemas

  • Resolvida uma falha com a bridge de plugins webkit na criação.
  • Resolvida uma falha com o reencaminhamento.
  • Resolvido um problema com a apresentação Speech em que esperava pela mensagem anterior terminar causando um utilização excessiva do CPU.
  • Resolvida uma falha com CFErrorGetDomain para estilos de apresentação Webkit.
  • Resolvida a listagem duplicada dos plugins de apresentação após reinstalar.
  • - Notification-Callback-Target é agora respeitado e utilizado, mesmo sem o cabeçalho do método.
  • Contornados os problemas com bases de dados de Histórico corruptas.
  • Resolvido um problema de ordenação ao eliminar uma entrada na lista de histórico.
